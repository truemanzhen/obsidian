# 消息过滤 — Tag 过滤与 SQL92 表达式

> 📄 源码路径：`filter/` + `broker/src/main/java/org/apache/rocketmq/broker/filter/`
> 🏷️ #核心类 #设计模式 #面试高频

## 🎯 核心思想

消息过滤允许 Consumer 只消费**符合条件的消息**，而不是所有消息。
RocketMQ 支持两种过滤方式：
1. **Tag 过滤**：基于消息的 Tag 属性，精确匹配
2. **SQL92 过滤**：基于 SQL92 表达式，支持复杂条件

## 📐 过滤架构

```
Producer                    Broker                     Consumer
  │                           │                           │
  │ send(msg, tag="order")    │                           │ subscribe("topic", "order || pay")
  │──────────────────────────→│                           │──────────────────────────→│
  │                           │                           │                           │
  │                           │ ① 存储消息                │ ② 注册订阅关系            │
  │                           │    Tag HashCode → CQ      │    SubscriptionData       │
  │                           │                           │                           │
  │                           │ ③ 拉取时过滤              │                           │
  │                           │    ConsumeQueue 级: Tag Hash│                          │
  │                           │    CommitLog 级: 完整属性   │                           │
  │                           │                           │                           │
  │                           │ ④ 返回匹配的消息           │                           │
```

## 📐 1. Tag 过滤

### 订阅关系构建

```java
// FilterAPI.buildSubscriptionData()
public static SubscriptionData buildSubscriptionData(String topic, String subString) {
    SubscriptionData subscriptionData = new SubscriptionData();
    subscriptionData.setTopic(topic);
    subscriptionData.setSubString(subString);  // "TagA || TagB || TagC"

    // 解析 Tag 集合
    if (!subString.equals(SubscriptionData.SUB_ALL)) {
        String[] tags = subString.split("\\|\\|");
        for (String tag : tags) {
            subscriptionData.getTags().add(tag.trim());
            // 计算 Tag 的 HashCode
            subscriptionData.getCodeSet().add(tag.trim().hashCode());
        }
    }
    return subscriptionData;
}
```

### 两级过滤

```java
// ExpressionMessageFilter.isMatchedByConsumeQueue()
// 第一级：ConsumeQueue 级过滤（Tag HashCode）
public boolean isMatchedByConsumeQueue(Long tagsCode, ConsumeQueueExt.CqExtUnit cqExtUnit) {
    if (ExpressionType.isTagType(subscriptionData.getExpressionType())) {
        if (subscriptionData.getSubString().equals(SubscriptionData.SUB_ALL)) {
            return true;  // 订阅所有
        }
        // 用 Tag HashCode 快速过滤
        return subscriptionData.getCodeSet().contains(tagsCode.intValue());
    }
}

// ExpressionMessageFilter.isMatchedByCommitLog()
// 第二级：CommitLog 级过滤（完整属性）
public boolean isMatchedByCommitLog(ByteBuffer msgBuffer, Map<String, String> properties) {
    if (ExpressionType.isTagType(subscriptionData.getExpressionType())) {
        // Tag 过滤在 ConsumeQueue 级已完成，这里直接通过
        return true;
    }
    // SQL92 过滤：执行表达式
    return doSQL92Filter(properties);
}
```

> [!important] Tag 过滤的优化
> Tag 过滤在 **ConsumeQueue 级** 就完成了，不需要读取 CommitLog：
> - ConsumeQueue 每条记录存储 Tag HashCode（8 字节）
> - 匹配 HashCode → 通过（可能有碰撞，但概率极低）
> - 不匹配 → 直接跳过，不需要读取完整消息
>
> 这是 RocketMQ 消息过滤**高性能**的关键。

### Tag 过滤的限制

```
Tag 过滤只支持：
  - 精确匹配：TagA
  - 多 Tag 或匹配：TagA || TagB || TagC
  - 订阅所有：* 或 SUB_ALL

不支持：
  - 范围查询
  - 模糊匹配
  - 复杂条件（AND、NOT、比较）
```

## 📐 2. SQL92 过滤

### 支持的语法

```sql
-- 基本比较
amount > 100
price >= 10.5 AND price <= 100.0
status = 'SUCCESS'

-- 字符串比较
topic = 'order' AND tags = 'pay'

-- IS NULL / IS NOT NULL
property1 IS NULL
property2 IS NOT NULL

-- IN
status IN ('SUCCESS', 'PENDING')

-- BETWEEN
amount BETWEEN 100 AND 1000

-- 逻辑组合
(status = 'SUCCESS' AND amount > 100) OR (status = 'PENDING' AND amount > 500)

-- 字符串函数
topic LIKE 'order%'
```

### SQL92 过滤实现

```java
// SqlFilter — SQL92 过滤器
public class SqlFilter implements FilterSpi {
    @Override
    public Expression compile(final String expr) throws MQFilterException {
        return SelectorParser.parse(expr);  // 解析 SQL92 表达式
    }

    @Override
    public String ofType() {
        return ExpressionType.SQL92;
    }
}

// SelectorParser — SQL92 解析器
// 使用 JavaCC 生成的词法/语法分析器
// 将 SQL92 表达式解析为 Expression 对象树
```

### BloomFilter 优化

```java
// ExpressionMessageFilter.isMatchedByConsumeQueue()
// SQL92 过滤在 ConsumeQueue 级使用 BloomFilter 预过滤

if (bloomDataValid) {
    // 使用 BloomFilter 快速判断消息是否可能匹配
    byte[] filterBitMap = cqExtUnit.getFilterBitMap();
    if (filterBitMap != null) {
        // BloomFilter 匹配 → 可能匹配，需要进一步检查
        // BloomFilter 不匹配 → 一定不匹配，直接跳过
        return bloomFilter.contains(consumerFilterData.getBloomFilterData(), filterBitMap);
    }
}
```

> [!tip] BloomFilter 的作用
> SQL92 过滤需要读取消息属性，开销较大。
> 用 BloomFilter 在 ConsumeQueue 级预过滤：
> - BloomFilter 说不匹配 → 一定不匹配 → 跳过
> - BloomFilter 说匹配 → 可能匹配 → 需要读取 CommitLog 验证
>
> 大大减少了需要读取 CommitLog 的消息数量。

## 📐 过滤流程图

```
Consumer 拉取消息请求：
  topic, queueId, offset, maxMsgNums, filterExpression

Broker 处理：
  ① 从 ConsumeQueue 读取索引
     │
     ▼
  ② ConsumeQueue 级过滤
     │
     ├─ Tag 过滤：比较 Tag HashCode
     │   match = subscriptionData.getCodeSet().contains(tagsCode)
     │
     └─ SQL92 过滤：BloomFilter 预过滤
         match = bloomFilter.contains(filterData, filterBitMap)
     │
     ▼
  ③ 通过 ConsumeQueue 过滤的消息 → 读取 CommitLog
     │
     ▼
  ④ CommitLog 级过滤（仅 SQL92）
     │
     ├─ Tag 过滤：已通过，直接返回
     │
     └─ SQL92 过滤：执行完整表达式
         match = expression.evaluate(properties)
     │
     ▼
  ⑤ 返回匹配的消息给 Consumer
```

## 📐 两种过滤方式对比

| 维度 | Tag 过滤 | SQL92 过滤 |
|------|---------|-----------|
| 语法 | `TagA \|\| TagB` | SQL92 表达式 |
| 过滤位置 | ConsumeQueue 级 | ConsumeQueue + CommitLog |
| 性能 | 极高（HashCode 比较） | 较高（BloomFilter + 表达式） |
| 灵活性 | 低（只能按 Tag） | 高（任意属性） |
| 存储开销 | 无额外开销 | BloomFilter 数据 |
| 适用场景 | 简单分类 | 复杂业务条件 |

## 💡 设计亮点

### 1. 两级过滤架构

```
ConsumeQueue 级（快，粗粒度）：
  - Tag HashCode 比较
  - BloomFilter 预过滤
  - 不需要读取消息体
  - 过滤掉大部分不匹配的消息

CommitLog 级（慢，精确）：
  - 完整 SQL92 表达式执行
  - 需要读取消息属性
  - 只处理 ConsumeQueue 级通过的消息

好处：大部分消息在 ConsumeQueue 级就被过滤掉了，
只有少量消息需要读取 CommitLog，性能极高。
```

### 2. BloomFilter 减少 I/O

```
BloomFilter 的特点：
  - 说"不匹配" → 100% 不匹配
  - 说"匹配" → 可能匹配（有假阳性）

用于 SQL92 过滤的预过滤：
  - BloomFilter 不匹配 → 直接跳过 → 不读 CommitLog
  - BloomFilter 匹配 → 读取 CommitLog 验证

大量减少 CommitLog 的随机读取。
```

### 3. 表达式编译缓存

```
SQL92 表达式只编译一次：
  SelectorParser.parse(expr) → Expression 对象

后续过滤直接使用编译后的 Expression 对象：
  expression.evaluate(properties)

避免重复解析，提高过滤性能。
```

## 📊 过滤性能对比

| 过滤方式 | ConsumeQueue 过滤 | CommitLog 过滤 | 总体性能 |
|---------|-------------------|---------------|---------|
| 不过滤 | 全部通过 | 不需要 | 最快 |
| Tag 过滤 | HashCode 比较 | 不需要 | 极快 |
| SQL92 + BloomFilter | BloomFilter 预过滤 | 表达式执行 | 较快 |
| SQL92 无 BloomFilter | 全部通过 | 表达式执行 | 慢 |

## 💡 学习收获

1. **两级过滤**：ConsumeQueue 粗过滤 + CommitLog 精确过滤
2. **Tag 过滤**：用 HashCode 比较，ConsumeQueue 级完成，性能极高
3. **SQL92 过滤**：支持复杂条件，用 BloomFilter 优化
4. **BloomFilter**：假阳性但无假阴性，减少 CommitLog 随机读取
5. **表达式编译缓存**：只编译一次，后续直接使用
