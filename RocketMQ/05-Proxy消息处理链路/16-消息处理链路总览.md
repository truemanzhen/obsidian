# 消息处理链路总览 — 从 gRPC 请求到消息存储

> 📄 源码路径：`proxy/src/main/java/org/apache/rocketmq/proxy/grpc/v2/`
> 🏷️ #架构设计 #核心类

## 📐 完整请求链路

```
Client (Producer)
    │
    │ gRPC: SendMessageRequest
    ▼
┌──────────────────────────────────────────────────────────────────┐
│ GrpcMessagingApplication (gRPC 服务层)                            │
│ ① Pipeline 预处理 (ContextInit → AuthN → AuthZ)                  │
│ ② 线程池路由 → ProducerThreadPool                                │
│ ③ 委托给 GrpcMessagingActivity                                   │
└────────────────────────┬─────────────────────────────────────────┘
                         ▼
┌──────────────────────────────────────────────────────────────────┐
│ DefaultGrpcMessagingActivity (Activity 路由层)                    │
│ 按操作类型分发给对应的 Activity                                    │
│                                                                  │
│ sendMessage → SendMessageActivity                                │
│ receiveMessage → ReceiveMessageActivity                          │
│ ackMessage → AckMessageActivity                                  │
│ queryRoute → RouteActivity                                       │
│ heartbeat → ClientActivity                                       │
│ ...                                                              │
└────────────────────────┬─────────────────────────────────────────┘
                         ▼
┌──────────────────────────────────────────────────────────────────┐
│ XxxActivity (业务逻辑层)                                          │
│ ① 参数校验 (validateTopic, validateConsumerGroup)                │
│ ② 消息转换 (gRPC Protobuf → 内部 Message 对象)                   │
│ ③ Queue 选择 (QueueSelector)                                     │
│ ④ 委托给 MessagingProcessor                                      │
└────────────────────────┬─────────────────────────────────────────┘
                         ▼
┌──────────────────────────────────────────────────────────────────┐
│ MessagingProcessor (处理器接口)                                   │
│ DefaultMessagingProcessor → 委托给 ProducerProcessor /           │
│                              ConsumerProcessor                   │
└────────────────────────┬─────────────────────────────────────────┘
                         ▼
┌──────────────────────────────────────────────────────────────────┐
│ ProducerProcessor / ConsumerProcessor                             │
│ ① 路由解析 (Topic → Broker 地址)                                 │
│ ② 构建 RemotingCommand (4.x 协议)                               │
│ ③ 调用 Broker 的实际接口                                         │
└────────────────────────┬─────────────────────────────────────────┘
                         ▼
┌──────────────────────────────────────────────────────────────────┐
│ Broker (存储层)                                                   │
│ CommitLog → ConsumeQueue → IndexFile                             │
└──────────────────────────────────────────────────────────────────┘
```

## 📐 类层次结构

```
                    GrpcMessagingActivity (接口)

                    DefaultGrpcMessagingActivity (实现)
                           │
                    ┌──────┴──────┐
                    │ 路由到各 Activity │
                    └──────┬──────┘
                           │
         ┌─────────────────┼─────────────────┐

    SendMessage      ReceiveMessage      AckMessage

         └─────────────────┼─────────────────┘
                           │
                   AbstractMessagingActivity
                   (公共校验逻辑)
                           │

                           │
              ┌────────────┼────────────┐
              │            │            │
       ProducerProcessor  ConsumerProcessor  ClientProcessor
              │            │

                    Broker Service Layer
```

## 📊 各 Activity 职责

| Activity | gRPC 方法 | 职责 |
|----------|----------|------|
| `SendMessageActivity` | `sendMessage` | 消息发送 |
| `ReceiveMessageActivity` | `receiveMessage` | Pop 消费（长轮询） |
| `AckMessageActivity` | `ackMessage` | 消息确认 |
| `ChangeInvisibleDurationActivity` | `changeInvisibleDuration` | 延长可见时间 |
| `ForwardMessageToDLQActivity` | `forwardMessageToDeadLetterQueue` | 转发到死信队列 |
| `RouteActivity` | `queryRoute` / `queryAssignment` | 路由查询 |
| `ClientActivity` | `heartbeat` / `notifyClientTermination` / `telemetry` | 客户端管理 |
| `EndTransactionActivity` | `endTransaction` | 事务提交/回滚 |
| `RecallMessageActivity` | `recallMessage` | 消息撤回 |

## 📐 设计模式总结

| 模式 | 在哪里 | 作用 |
|------|--------|------|
| **委托模式** | `DefaultGrpcMessagingActivity` → 各 `Activity` | 按操作类型分发 |
| **模板方法** | `AbstractMessagingActivity` | 公共校验逻辑 |
| **策略模式** | `QueueSelector` | 不同场景的队列选择策略 |
| **异步组合** | `CompletableFuture` 链式调用 | 全链路异步 |
| **转换器模式** | `GrpcConverter` | Protobuf ↔ 内部对象转换 |

## 📊 消息发送 vs 消息消费对比

```
发送链路：
Producer → gRPC → Pipeline → Thread Pool → SendMessageActivity
    → buildMessage() → QueueSelector → MessagingProcessor
    → ProducerProcessor → Broker.putMessage() → CommitLog

消费链路：
Consumer → gRPC → Pipeline → Thread Pool → ReceiveMessageActivity
    → buildFilter() → QueueSelector → MessagingProcessor
    → ConsumerProcessor → Broker.popMessage() → ConsumeQueue → CommitLog
    → 长轮询等待 → 返回消息
```
