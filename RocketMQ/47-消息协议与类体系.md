# 消息协议与类体系

> 源码位置：`common/src/main/java/org/apache/rocketmq/common/message/`

## 🏗️ 类继承体系

RocketMQ 的消息模型通过类继承逐层扩展，从客户端到 Broker 存储层层递进。

```
Message (客户端消息)
  └── MessageExt (扩展信息：Broker/Queue/Offset)
        ├── MessageClientExt (客户端扩展：用于客户端传输)
        └── MessageExtBrokerInner (Broker 内部：编码后的完整消息)
              └── MessageExtBatch (批量消息)

MessageBatch (批量消息容器，extends Message, implements Iterable)
```

## 📦 各类详解

### 1. Message（基础消息）

```java
public class Message implements Serializable {
    private String topic;                    // Topic
    private int flag;                        // 标志位
    private Map<String, String> properties;  // 属性（Tags/Keys/自定义）
    private byte[] body;                     // 消息体
    private String transactionId;            // 事务 ID

    // 便捷方法
    public void setTags(String tags) {
        putProperty(MessageConst.PROPERTY_TAGS, tags);
    }
    public void setKeys(String keys) {
        putProperty(MessageConst.PROPERTY_KEYS, keys);
    }
    public void putUserProperty(String name, String value) {
        properties.put(name, value);
    }
}
```

### 2. MessageExt（扩展消息）

## 📊 消息编码格式

### CommitLog 中的消息格式

```
┌───────────────────────────────────────────────────────────┐
│                 MessageExtBrokerInner 编码                 │
├──────────┬────────────────────────────────────────────────┤
│ 4 bytes  │ TOTAL_SIZE（总大小）                            │
│ 4 bytes  │ MAGIC_CODE（魔数，-626843481）                  │
│ 4 bytes  │ BODY_CRC（消息体 CRC32）                        │
│ 4 bytes  │ QUEUE_ID（队列 ID）                             │
│ 4 bytes  │ FLAG（标志位）                                  │
│ 8 bytes  │ QUEUE_OFFSET（队列偏移量）                      │
│ 8 bytes  │ PHYSICAL_OFFSET（物理偏移量/CommitLog Offset）   │
│ 4 bytes  │ SYS_FLAG（系统标志：压缩/事务/批量）             │
│ 8 bytes  │ BORNTIMESTAMP（创建时间）                       │
│ 16 bytes │ BORNHOST（创建主机 IP+Port）                    │
│ 8 bytes  │ STORETIMESTAMP（存储时间）                      │
│ 16 bytes │ STOREHOST（存储主机 IP+Port）                   │
│ 4 bytes  │ RECONSUME_TIMES（重消费次数）                   │
│ 8 bytes  │ PREPARED_TRANSACTION_OFFSET（事务偏移量）       │
│ 4 bytes  │ BODY_LENGTH（消息体长度）                       │
│ N bytes  │ BODY（消息体）                                  │
│ 2 bytes  │ TOPIC_LENGTH（Topic 长度）                      │
│ M bytes  │ TOPIC（Topic 名称）                             │
│ 2 bytes  │ PROPERTIES_LENGTH（属性长度）                   │
│ K bytes  │ PROPERTIES（属性，key=value\x01\x02 格式）      │
└──────────┴────────────────────────────────────────────────┘
```

### sysFlag 标志位

```java
// Bit 0: 是否压缩
MessageSysFlag.COMPRESSION_FLAG = 0x1;

// Bit 1-2: 压缩类型
MessageSysFlag.COMPRESSION_LZ4_TYPE  = 0x1 << 1;  // 0x2
MessageSysFlag.COMPRESSION_ZSTD_TYPE = 0x2 << 1;  // 0x4
MessageSysFlag.COMPRESSION_ZLIB_TYPE = 0x3 << 1;  // 0x6

// Bit 3: 是否事务消息
MessageSysFlag.TRANSACTION_PREPARED_TYPE = 0x1 << 3;  // 0x8

// Bit 4: 是否批量消息
MessageSysFlag.MULTI_MESSAGES_FLAG = 0x1 << 4;  // 0x10
```

## 🔄 属性编码格式

```
properties 格式：
key1=value1\x01key2=value2\x01key3=value3\x02

\x01 = NAME_VALUE_SEPARATOR（属性分隔符）
\x02 = PROPERTY_SEPARATOR（键值对分隔符）
```

## 💡 设计亮点

1. **分层继承**：Message → MessageExt → MessageExtBrokerInner，职责清晰
2. **属性系统**：通过 properties Map 实现丰富的元数据传递
3. **sysFlag 位图**：用 bit 标记压缩、事务、批量等特性，高效
4. **向后兼容**：MAGIC_CODE 区分 V1/V2 版本
5. **Tag 哈希**：tagsCode 用于 ConsumeQueue 快速过滤，避免读取消息体

## 🔗 与其他模块的关系

- **CommitLog**：消息编码写入 CommitLog
- **ConsumeQueue**：存储 tagsCode + offset + size（20 字节）
- **SendMessageProcessor**：构建 MessageExtBrokerInner
- **MessageDecoder**：解码消息
