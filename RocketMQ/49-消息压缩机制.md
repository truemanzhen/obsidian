# 消息压缩机制

> 源码位置：`common/src/main/java/org/apache/rocketmq/common/compression/`

## 🏗️ 为什么需要压缩

消息体压缩可以：
- **减少存储空间**：节省磁盘
- **减少网络传输**：降低带宽消耗
- **提高吞吐量**：压缩后的消息传输更快

## 📊 支持的压缩算法

| 算法 | 压缩比 | 压缩速度 | 解压速度 | 适用场景 |
|---|---|---|---|---|
| **LZ4** | 2.10 | 740 MB/s | 4500 MB/s | ⭐ 默认，速度优先 |
| **Zstd** | 2.89 | 530 MB/s | 1700 MB/s | 压缩比优先 |
| **Zlib** | 2.74 | 95 MB/s | 400 MB/s | 兼容性优先 |

## 📦 核心类

| 类 | 职责 |
|---|---|
| `Compressor` | 压缩器接口 |
| `Lz4Compressor` | LZ4 压缩实现 |
| `ZstdCompressor` | Zstd 压缩实现 |
| `ZlibCompressor` | Zlib 压缩实现 |
| `CompressorFactory` | 压缩器工厂 |
| `CompressionType` | 压缩类型枚举 |

## 📝 Compressor 接口

```java
public interface Compressor {
    byte[] compress(byte[] src, int level) throws IOException;
    byte[] decompress(byte[] src) throws IOException;
}
```

## 🔄 压缩流程

### 发送端（Producer）

```
消息体较大（超过阈值）
    ↓
MessageSysFlag.setCompressionType(sysFlag, CompressionType.LZ4)
    ↓
CompressorFactory.getCompressor(CompressionType.LZ4)
    ↓
compressor.compress(messageBody, level)
    ↓
设置 sysFlag 标记压缩类型
    ↓
发送压缩后的消息体
```

### Broker 端（存储）

```
Broker 收到消息
    ↓
检查 sysFlag 是否压缩
    ├── 已压缩 → 直接存储压缩后的消息体
    └── 未压缩 → 可选：Broker 侧压缩
    ↓
写入 CommitLog
```

### 消费端（Consumer）

```
消费者拉取消息
    ↓
检查 sysFlag 压缩标志
    ├── 已压缩 → 解压消息体
    │   └── CompressorFactory.getCompressor(type).decompress(body)
    └── 未压缩 → 直接使用
    ↓
返回原始消息体
```

## 🔧 CompressorFactory

```java
public class CompressorFactory {
    public static Compressor getCompressor(CompressionType type) {
        switch (type) {
            case LZ4:
                return new Lz4Compressor();
            case ZSTD:
                return new ZstdCompressor();
            case ZLIB:
                return new ZlibCompressor();
            default:
                throw new RuntimeException("Unsupported compression type: " + type);
        }
    }
}
```

## 🔧 sysFlag 标记

```java
// MessageSysFlag 中的压缩相关位
// Bit 0: 是否压缩
COMPRESSION_FLAG = 0x1;

// Bit 1-2: 压缩类型
COMPRESSION_LZ4_TYPE  = 0x1 << 1;  // 0x2
COMPRESSION_ZSTD_TYPE = 0x2 << 1;  // 0x4
COMPRESSION_ZLIB_TYPE = 0x3 << 1;  // 0x6

// 设置压缩标志
int sysFlag = 0;
sysFlag |= MessageSysFlag.COMPRESSION_FLAG;  // 标记已压缩
sysFlag |= CompressionType.LZ4.getCompressionFlag();  // 标记压缩类型

// 读取压缩类型
boolean compressed = (sysFlag & MessageSysFlag.COMPRESSION_FLAG) != 0;
int typeValue = (sysFlag >> 1) & 0x3;
CompressionType type = CompressionType.findByValue(typeValue);
```

## 💡 设计亮点

1. **工厂模式**：CompressorFactory 统一创建压缩器
2. **sysFlag 标记**：压缩类型编码在 sysFlag 中，不额外存储
3. **多种算法**：LZ4/Zstd/Zlib 满足不同需求
4. **透明解压**：消费者无感知，自动根据 sysFlag 解压
5. **可配置**：通过配置选择默认压缩算法

## ⚠️ 注意事项

1. **压缩阈值**：消息体太小压缩反而增大（建议 > 1KB）
2. **CPU 开销**：压缩会消耗 CPU，高吞吐场景注意
3. **压缩级别**：level 参数影响压缩比和速度的平衡
4. **版本兼容**：老版本不支持 Zstd，需要升级

## 🔗 与其他模块的关系

- **CommitLog**：压缩后的消息体写入 CommitLog
- **MessageDecoder**：解码时处理压缩标志
- **Producer**：发送时压缩
- **Consumer**：消费时解压
